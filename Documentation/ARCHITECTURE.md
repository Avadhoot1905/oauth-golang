## ğŸ¯ Request/Response Flow Details

### /authorize Endpoint
```
INPUT (API):
  - Query Params: client_id, redirect_uri, response_type, state, 
                  code_challenge, code_challenge_method

PROCESS:
  1. Validate client_id (DB: client_repo)
  2. Validate redirect_uri
  3. Validate PKCE parameters
  4. Generate session ID
  5. Store session (Memory: authcode_service)
  6. Redirect to Google OAuth

OUTPUT (API):
  - 302 Redirect to Google
```

### /callback Endpoint
```
INPUT (Google OAuth):
  - Query Params: code, state

PROCESS:
  1. Retrieve session by state (Memory: authcode_service)
  2. Exchange code with Google (HTTP: Google API)
  3. Get user info from Google (HTTP: Google API)
  4. Generate authorization code
  5. Store auth code + user info (Memory: authcode_service)
  6. Build redirect URL

OUTPUT (API):
  - 302 Redirect to client with code & state
```

### /token Endpoint
```
INPUT (API):
  - Body: grant_type, code, redirect_uri, client_id, 
          client_secret, code_verifier

PROCESS:
  1. Validate grant_type
  2. Validate client credentials (DB: client_repo)
  3. Retrieve auth code (Memory: authcode_service)
  4. Verify PKCE code_verifier
  5. Create/update user (DB: user_repo)
  6. Generate tokens (JWT)
  7. Store refresh token (DB: token_repo)
  8. Delete auth code

OUTPUT (API):
  - JSON: access_token, refresh_token, id_token, expires_in
```

### /userinfo Endpoint
```
INPUT (API):
  - Header: Authorization: Bearer <token>

PROCESS:
  1. Extract token from header
  2. Verify JWT signature & expiration
  3. Get user from database (DB: user_repo)

OUTPUT (API):
  - JSON: User information (sub, email, name, etc.)
```

### /introspect Endpoint
```
INPUT (API):
  - Body: token, token_type_hint

PROCESS:
  1. Verify JWT signature & expiration
  2. Check if token is revoked (DB: token_repo)
  3. Build introspection response

OUTPUT (API):
  - JSON: active, scope, client_id, username, exp, iat, sub
```
